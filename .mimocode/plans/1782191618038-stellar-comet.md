# Plan: Custom Section Components + Code Coloring

## Goal
Add reusable CSS utility classes for agents injecting custom HTML, plus lightweight theme-aware syntax highlighting via Prism.js.

## Context
When agents use `type = "custom"` or `type = "text"` sections, they inject raw HTML. Currently there are no pre-built CSS classes for common patterns like callouts, keyboard shortcuts, or styled dividers. Agents must write inline styles or guess at what CSS variables exist.

## What Agents Need
Agents injecting custom HTML commonly need:
- **Callout/alert boxes** — info, warning, success, error notices
- **Keyboard shortcut display** — `<kbd>` styling
- **Themed dividers** — styled `<hr>` using theme colors
- **Syntax highlighting** — for code blocks beyond plain `<pre><code>`

## Design

### 1. Prism.js for Code Coloring (~5KB gzipped)

**Why Prism.js over highlight.js:**
- 5KB core vs 14KB for highlight.js
- Built-in Dracula-compatible CSS via `.token.*` classes
- Official line-number plugin available
- CDN: `https://cdnjs.cloudflare.com/ajax/libs/prism/1.29.0/`
- Supports: Python, JS, HTML, CSS, TOML, Bash (6 languages = ~5KB)

**Implementation:**
- Add Prism CSS + JS from CDN in `<head>`
- Add a Dracula-themed CSS override using our `--*` variables
- Update `code_block` section type to auto-detect language or use a `language` field
- Wrap `<pre><code>` with language class for Prism to pick up

**CSS override (Dracula-themed):**
```css
.token.comment { color: var(--muted); }
.token.keyword { color: var(--pink); }
.token.string { color: var(--yellow); }
.token.function { color: var(--green); }
.token.number { color: var(--orange); }
.token.operator { color: var(--pink); }
.token.punctuation { color: var(--fg); }
.token.class-name { color: var(--cyan); }
.token.boolean { color: var(--orange); }
.token.builtin { color: var(--cyan); }
.token.decorator { color: var(--purple); }
```

### 2. Utility CSS Classes for Custom HTML

Add these classes to the template CSS, all using `var(--*)` for theme awareness:

#### `.callout` — admonition boxes
```html
<div class="callout callout-info">ℹ️ Note: This is informational.</div>
<div class="callout callout-warning">⚠️ Warning: Be careful.</div>
<div class="callout callout-success">✅ Success: It worked!</div>
<div class="callout callout-error">❌ Error: Something broke.</div>
```

CSS:
```css
.callout {
  padding: 12px 16px;
  border-radius: 8px;
  border-left: 4px solid var(--muted);
  background: color-mix(in srgb, var(--panel) 50%, transparent);
  margin: 12px 0;
  font-size: 0.85rem;
}
.callout-info { border-left-color: var(--cyan); }
.callout-warning { border-left-color: var(--orange); }
.callout-success { border-left-color: var(--green); }
.callout-error { border-left-color: var(--red); }
```

#### `.kbd` — keyboard shortcut display
```html
Press <kbd class="kbd">Ctrl</kbd> + <kbd class="kbd">K</kbd> to search.
```

CSS:
```css
.kbd {
  display: inline-block;
  padding: 2px 6px;
  font-size: 0.75rem;
  font-family: var(--font);
  background: var(--panel);
  border: 1px solid var(--muted);
  border-radius: 4px;
  box-shadow: 0 1px 2px rgba(0,0,0,0.1);
}
```

#### `.divider` — themed horizontal rule
```html
<hr class="divider">
```

CSS:
```css
.divider {
  border: none;
  border-top: 1px solid var(--muted);
  margin: 24px 0;
}
```

### 3. Update `code_block` Section Type

Add optional `language` field to TOML schema:
```toml
[[sections]]
id = "quickstart"
type = "code_block"
title = "Quick Start"
icon = "play"
language = "bash"
code = "uv sync"
```

Template renders as:
```html
<pre><code class="language-bash">{{ section.code | e }}</code></pre>
```
